Getting started

Bookmarks.dev is a bookmark manager created and designed with Developers & Co in mind. Take a look at our About page for more details.

Create an Account first. You will be taken to the Login/Registration form, where you can choose to connect via email and password, or via one of the many supported identity providers ( Github, Google, Twitter, StackOverflow, LinkedIn or Facebook). Choose an username, add your first bookmark and you are good to go.

Save bookmarks

We've made it easy to save bookmarks:

  • The first option is with the help of bookmarklets. Bookmarklets are little javascript links that live in the bookmarks toolbar of your browser. Just drag one of the links bellow to your browser toolbar:
    • +Bookmarks.dev - popup - when clicked a popup to save the current URL is displayed [recommended for desktops and notebooks with good resolution]
    • +Bookmarks.dev - go back - when clicked it redirects you to the New Bookmark form. Once added you go back to the initial page [recommended for phones or smaller resolution devices]

    You can select the text on page that you want to be added to the description of the bookmark before you click the bookmarklet's link

  • Save directly using the button from the navigation menu
  • Use the Save to bookmarks.dev chrome extension for Chromium based browsers - same behaviour as the first bookmarklet

Public & Private bookmarks

When saving a bookmark you have the option to make it public - this won't be available if you opt in for the all private modus (soon to be implemented). Please share only dev related links that others might benefit from - SPAM will not be tolerated and you risk having your account blocked

The public bookmarks are regularly published on Github at https://github.com/CodepediaOrg/bookmarks

Search bookmarks

You can search bookmarks by placing your search terms in the Search bar. You can choose to search in Public Bookmarks or in Mine & Favorites, by selecting one of the options beneath the Search Bar. Tags must be embedded in square bracktes - e.g. [java].


Let's say you want to look for personal bookmarks that contain the text exception and are tagged with java. Then would put exception [java] in the search bar, and use the Personal & Favorites option

You have the possibility to save this search by clicking the Save search button. This way it will be displayed in the auto completion when you input text in the Search bar

Search via query parameters

You can execute the same search via query parameters

You can trigger the same search as above by calling the following URL: https://www.bookmarks.dev/search?q=exception [java]&sd=personal
  • q - here is your input search text
  • sd - search domain with two options
    • personal - mine and favorite bookmarks
    • public - public bookmarks (may include personal bookmarks if they come from same user)

This is very handy if you use Web Search tools where you can define custom queries, like Alfred Web Search if you are on Mac
